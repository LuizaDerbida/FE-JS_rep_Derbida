У міру зростання популярності Інтернету в дев'яностих роках, також зростав намір застосовувати до сайту певний дизайн. Веб-розробники покладалися на конкретні теги HTML для покращення веб-сторінок:

  • <basefont> визначав шрифт всього HTML-документа;
  • <font> визначав гарнітуру, колір та розмір тексту, що знаходиться всередині;
  • <center> вирівнював по центру весь свій вміст;
  • <big> збільшував розмір тексту;
  • <strike> відображав текст перекресленим.
Також могли бути використані кілька атрибутів HTML:
  • <bgcolor> визначав колір елемента фону;
  • <text> визначав колір тексту;
  • атрибути margin могли бути використані для додавання місця з будь-якої сторони елемента.

Навіщо уникати таблиць?

Але перш за все, для створення колонок, візуального вирівнювання елементів і, загалом, позиціонування елементів щодо один одного, веб-розробники використовували елемент <table> для дизайну своїх веб-сторінок, тому що він природно забезпечував візуальну сітку . Такий підхід був громіздким з кількох причин:
  • таблиці в HTML багатослівні : вони вимагають багато шаблонного коду;
  • розмітка була семантично неправильною : таблиці повинні використовуватися для багатовимірних даних;
  • зміна макета вимагала зміни розмітки: якщо ми хотіли перемістити ліву колонку праворуч, то мали модифікувати структуру HTML ;
  • таблиці були схильні до помилок синтаксису : рядки та комірки для коректності повинні йти та вкладатися певним чином;
  • розмітка була нечитаемой : таблиці вкладалися інші таблиці, щоб отримати додаткові колонки всередині колонок.
Саме тому від застосування таблиць як інструмент верстки повільно відмовилися і замість них був використаний CSS.

Що таке CSS

CSS (Cascading Style Sheets) означає каскадні таблиці стилів і є мовою розмітки стилів (як HTML або XML). Таким чином, CSS нічого не представляє сам собою, якщо не пов'язаний з HTML-документом. CSS привносить життя HTML-документ, вибираючи шрифти, застосовуючи кольори, визначаючи відступи, позиціонуючи елементи, анімуючи взаємодії та багато іншого.

Як працює CSS

CSS вибирає елемент HTML (наприклад, абзац), задає властивість для зміни (наприклад, колір) і застосовує певне значення (наприклад, червоний) - p { color: red;}

Де я можу писати CSS?

CSS як атрибут

Ви можете писати CSS безпосередньо в елементах HTML за допомогою атрибуту style: p style="color: red;

CSS у head

Ви можете використовувати тег style усередині head вашого HTML-документу:<head> <style> p { color: red;} </style> </head>

CSS в окремому файлі

Ви можете писати свій CSS в окремому файлі з розширенням .css, а потім зв'язати його з HTML за допомогою тега <link> . <link rel="stylesheet" href="style.css"> Даний HTML-документ, який «викликає» файл CSS, у нашому випадку файл з іменем style.css розташований у тій же папці, що й CSS-файл. Цей третій метод , що використовує окремий файл CSS, є кращим .
